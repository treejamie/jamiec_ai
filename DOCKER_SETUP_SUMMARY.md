# Docker Setup Summary

## Files Created

### 1. Dockerfile
Multi-stage Docker build configuration that:
- Uses Elixir 1.18.3 with Erlang/OTP 27.2
- Builds for AMD64 and ARM64 architectures
- Compiles assets (Tailwind + esbuild)
- Creates optimized release build
- Runs as non-root user (`nobody`)

**Location**: `/Dockerfile`

### 2. .dockerignore
Excludes unnecessary files from Docker build context:
- Git files
- Build artifacts
- Test files
- Documentation
- Editor configs
- Local environment files

**Location**: `/.dockerignore`

### 3. GitHub Actions Workflow
CI/CD pipeline that triggers on push to `main` branch:
- Sets up QEMU for ARM64 emulation
- Uses Docker Buildx for multi-architecture builds
- Builds for linux/amd64 and linux/arm64
- Pushes to GitHub Container Registry (ghcr.io)
- Tags with: `latest`, `main`, and `sha-<commit>`
- Uses GitHub Actions cache for faster builds

**Location**: `/.github/workflows/docker-build.yml`

### 4. Release Configuration
Generated by `mix phx.gen.release`:
- `rel/overlays/bin/server` - Server startup script
- `rel/overlays/bin/migrate` - Database migration script
- `lib/jamiec/release.ex` - Release helper module for migrations

### 5. Documentation
- `DEPLOYMENT.md` - Complete deployment guide
- `DOCKER_SETUP_SUMMARY.md` - This file

## How It Works

### Development
1. Push code to any branch
2. CI runs tests (existing `.github/workflows/ci.yml`)

### Production Deployment
1. Merge code to `main` branch
2. GitHub Actions automatically:
   - Builds Docker images for AMD64 + ARM64
   - Pushes to `ghcr.io/jamiec/jamiec`
   - Tags as `latest` and `sha-<commit>`
3. Coolify on Raspberry Pi:
   - Pulls pre-built ARM64 image
   - Runs migrations (if configured)
   - Starts new container
   - No compilation on Pi needed!

## Required Environment Variables

Set these in Coolify:

```bash
SECRET_KEY_BASE=<generate with: mix phx.gen.secret>
DATABASE_URL=postgresql://user:pass@host/database
PHX_HOST=yourdomain.com
PHX_SERVER=true
PORT=4000
POOL_SIZE=10
```

## Image Registry

Images are pushed to GitHub Container Registry:
- Registry: `ghcr.io`
- Image: `ghcr.io/jamiec/jamiec:latest`

**Make package public**:
1. Go to: https://github.com/users/jamiec/packages
2. Find the `jamiec` package
3. Package settings → Change visibility → Public

## Testing Locally

### Build and test:
```bash
# Build for AMD64 (your machine)
docker build --platform linux/amd64 -t jamiec:test .

# Run locally
docker run --rm \
  -e SECRET_KEY_BASE=$(mix phx.gen.secret) \
  -e DATABASE_URL=postgresql://localhost/jamiec_dev \
  -e PHX_HOST=localhost \
  -e PHX_SERVER=true \
  -e PORT=4000 \
  -p 4000:4000 \
  jamiec:test
```

### Test multi-arch build:
```bash
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  -t jamiec:multi-arch \
  .
```

## First Deployment Steps

1. **Merge this PR to main** - triggers first Docker build

2. **Wait for GitHub Actions** - check Actions tab for build status

3. **Make package public** (if needed):
   - Go to GitHub Packages
   - Find your `jamiec` package
   - Change visibility to Public

4. **Configure Coolify**:
   - Service Type: Docker Image
   - Registry: ghcr.io
   - Image: `jamiec/jamiec:latest`
   - Add environment variables (see above)
   - Set pull policy to "Always"

5. **Run migrations** (first time only):
   - In Coolify console: `/app/bin/migrate`
   - Or configure as pre-deployment command

6. **Deploy** - Coolify pulls and starts the container

## Benefits

✅ **No compilation on Pi** - Pre-built images pulled from registry
✅ **Fast deployments** - Just pull and restart container
✅ **Multi-arch support** - Can test locally on AMD64, deploy to ARM64
✅ **Version control** - Specific commit SHAs for rollbacks
✅ **Automated** - Push to main = automatic deployment
✅ **Secure** - Non-root container, secrets via environment

## Rollback

If something goes wrong:
1. In Coolify, change image tag to previous SHA
2. Click "Redeploy"
3. Or revert the commit in git and push
